2. Notes

2.1. CLI Pastebins

Here are several paste-bin sites that can be used from the CLI without having to download anything. All that is needed is a network connection and cURL or wget. 

## More information can be found at
## https://transfer.sh/
$ =curl --upload-file= ./hello.txt https://transfer.sh/hello.txt
https://transfer.sh/66nb8/hello.txt

## https://unix.stackexchange.com/questions/108493/easy-way-to-paste-command-line-output-to-paste-bin-services/108495#108495
$ <command> | curl -F 'sprunge=<-' http://sprunge.us

## OR

$ <command> 2>&1 | curl -F 'f:1=<-' ix.io

## OR download from ix.io
$ curl ix.io/client > /path/to/ix
$ chmod 755 /path/to/ix
$ cat FILE | /path/to/ix 

## from https://cfp.vim-cn.com/
$ <command> | curl -F 'vimcn=<-' https://cfp.vim-cn.com/
https://cfp.vim-cn.com/cbc

2.2. File Manipulation

2.2.1. Shell based transformations

  1. Finding and transforming a huge collection of MP3s

    I have about 4335 MP3s that I got from various sources and wanted to clean them up, so that the file names would be consistent, as well as the ID3 tags. This is how I went about doing that. 

    bash$ find . -name "*.mp3" | wc -l
4335

    First thing is to rename the directories and files, so that all spaces in the names are replaced with an underscore (_). 

    ## verify with 'rename -n'
bash$ find . -name "* *" -type d -print0 | xargs -0 rename -n 's/ /_/g'

## now actually do the work
bash$ find . -name "* *" -type d -print0 | xargs -0 rename -n 's/ /_/g'

    I still had some directories though that did not get converted. This was because some of the parent directories got transformed first, and so the list of directories provided by find was invalid for the sub-directories. Not a problem, just re-run the above command a couple of times to get everything.

    Now the files themselves. 

    bash$ find . -name "* *" -type f -print0 | xargs -0 rename -n 's/ /_/g'

bash$ find . -name "* *" -type f -print0 | xargs -0 rename -n 's/ /_/g'

    Now I need to clean up the filenames, as they contain (, ), [, ] as well as the names of various websites that they were downloaded from. 

    bash$ find . -name "*www.downloadming.com*" -type f -print0 | xargs -0 rename 's/_-_www.downloadming.com//g'

bash$ find . -name "*Gana.PK*" -type f -print0 | xargs -0 rename 's/\[Gana.PK\]//g'

bash$ find . -name "*SongsPK*" -type f -print0 | xargs -0 rename 's/\(www.SongsPK.info\)//g'

bash$ find . -iname "*Songs*PK*" -type f -print0 | xargs -0 rename 's/\(www.songs.pk\)//g'

bash$ find . -iname "*original*soundtrack*" -type f -print0 | xargs -0 rename 's/\[Original_Soundtrack\]//g'

bash$ find . -iname "*webmusic.in*" -type f -print0 | xargs -0 rename 's/\(webmusic.in\)//g'

bash$ find . -iname "*.pk*" -type f -print0 | xargs -0 rename 's/\(SongsPK.info\)//g'

bash$ find . -iname "*SongsPK*" -type f -print0 | xargs -0 rename 's/\(SongsPK.info\)//g'

    Is there a better way that I could have done this ? I'm sure there is, but I was using the kiss principle. As one of my CS professors used to say:

    Start vanilla and than get all fancy.

    Or something along those lines.

    There is a lot more to do still, like how to edit the ID3 tags from the command line, but we'll get to that later.
